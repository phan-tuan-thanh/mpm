# Plan: Task 6.3 — Implement DTOs và validation

## Task Info
- **Task ID**: 6.3
- **Spec**: user-authentication
- **Wave**: 6 (depends on 6.1, 6.2 being in progress)

## Approach
Tạo 3 DTO files cho auth module sử dụng class-validator decorators:
1. `AuthCallbackDto` — validate OAuth callback params (code, state)
2. `LoginResponseDto` — response shape cho login/callback endpoint
3. `SessionListDto` — response shape cho session list endpoint

## Files to Create
- `apps/backend/src/auth/dto/auth-callback.dto.ts`
- `apps/backend/src/auth/dto/login-response.dto.ts`
- `apps/backend/src/auth/dto/session-list.dto.ts`

## Acceptance Criteria (from Requirements)
- **1.2**: Auth_Client gửi state parameter cùng authorization code
- **1.9**: State parameter trong callback phải khớp với giá trị đã lưu
- **2.4**: Session list trả về session ID, device info, IP, createdAt, lastActivity, isCurrent flag

## Dependencies
- `@mpm/shared-types` — SessionData interface
- `class-validator` — validation decorators
- `class-transformer` — transformation decorators (if needed)

## Validation Rules
- AuthCallbackDto:
  - `code`: @IsString(), @IsNotEmpty()
  - `state`: @IsString(), @IsNotEmpty()
- LoginResponseDto:
  - `accessToken`: string field
- SessionListDto:
  - `sessions`: array of SessionInfoDto (sessionId, deviceInfo, ipAddress, createdAt, lastActivity, isCurrent)
